Features

核心特性

从触发到执行，从可视化编辑到脚本扩展，覆盖完整的自动化链路。

多类型触发器

Cron 定时、系统事件（开机/USB/网络/进程）、课表触发、手动按钮。覆盖教室与日常全场景。

20 种动作类型

应用启动、媒体控制、系统电源、Toast 通知、控制流（if/loop/变量）、Lua 脚本，开箱即用。

可视化积木编辑

基于 React Flow 的三栏编辑器，拖拽积木块组合动作链，支持条件分支、循环、变量传递。

Lua 脚本扩展

内置 mlua 引擎（Lua 5.4），严格沙箱隔离，支持脚本市场直连 GitHub 仓库浏览与安装。

容错与日志

每动作可配容错策略（继续/停止/回滚/通知），执行日志持久化到 SQLite，便于回溯。

便携式部署

数据与程序同目录，NSIS 安装包分发，支持 U 盘导入导出配置，适配学校机器环境。

动作链执行引擎的精华：递归遍历、容错策略、日志记录，一个不少。

ChainEngine

动作链执行引擎是 Dominate 的核心调度逻辑。它从数据库加载 Flow 的所有 actions，按 parent_id 构建树形结构，递归执行根节点。

  • ▸ 递归执行：按 parent_id 与 order 排序，支持嵌套动作链
  • ▸ 控制流：if/else 分支、loop 循环、break/continue 信号
  • ▸ 容错策略：Continue / Stop / Rollback / Notify
  • ▸ 日志持久化：每动作执行结果写入 SQLite execution_logs

/// 执行整条快捷指令
///
/// 加载 flow 的所有 actions 并按顺序执行，记录执行日志。
pub fn execute_flow(&self, flow_id: &str) -> Result<ExecutionLog> {
    let repo = Repository::new(&self.db);

    let flow = repo.get_flow(flow_id)?
        .ok_or_else(|| AppError::NotFound(format!("快捷指令 {} 不存在", flow_id)))?;

    if !flow.enabled {
        return Err(AppError::Trigger(format!(
            "快捷指令 {} 已禁用，无法执行", flow_id
        )));
    }

    let actions = repo.list_actions(flow_id)?;
    tracing::info!(
        "开始执行快捷指令: id={} name={} actions={}",
        flow_id, flow.name, actions.len()
    );

    let mut ctx = ExecutionContext::new(
        flow_id.to_string(),
        self.app_handle.clone(),
        self.global_variables.clone(),
    );

    let mut flow_log = ExecutionLog::start(flow_id.to_string(), None);
    repo.insert_log(&flow_log)?;

    // 递归执行根节点（parent_id = None）
    let result = self.execute_children(
        &actions, None, &mut ctx, &repo, flow.default_fault_strategy
    );

    match &result {
        Ok(_) => {
            flow_log.succeed();
            tracing::info!("快捷指令执行成功: id={}", flow_id);
        }
        Err(e) => {
            flow_log.fail(e.to_string());
            tracing::error!("快捷指令执行失败: id={} err={}", flow_id, e);
        }
    }

    repo.insert_log(&flow_log)?;
    Ok(flow_log)
}

FAQ

常见问题

Dominate 是什么？ +

Dominate 是一款面向 Windows 的桌面自动化管理工具，定位为"个人自动化助手"。借鉴 iOS 快捷指令设计理念，通过拖拽积木块组合动作链，配合多种触发器实现日常场景的自动化管理。

支持哪些操作系统？ +

仅支持 Windows 10/11，x64 架构。需要全局管理员权限运行（UAC 已关闭）。不考虑跨平台，专注于 Windows 生态深度集成。

项目是开源的吗？ +

完全开源，基于 MIT 协议发布。源代码托管在 GitHub，欢迎 Star、Issue、PR。

Lua 脚本安全吗？ +

默认严格沙箱，禁用 os.execute / io.popen / loadfile 等危险 API。用户可在设置中开启可选宽松沙箱（自负风险）。所有 Lua 脚本默认 10 秒超时，防卡死。

数据存储在哪里？是否便携？ +

数据存储在程序安装目录下的 data/dominate.db（SQLite）。便携式部署，支持 U 盘导入导出配置（.dominate 格式，含 JSON 数据 + Lua 脚本）。

当前进度如何？ +

Phase 1（核心调度 + 动作执行）已完成，15 个 unit tests 全部通过。Phase 2（UI 骨架 + 首页 Dashboard）开发中。完整 6 阶段计划见 SPEC.md。
